# Purpose

The purpose of the files in the `technical` folder is to document a process for using AI to help assist in building this 
application.  

## History

My process is an iterative approach for building spec files.  Iterations are creating or editing a spec, asking agents
to create responses and repeating.  This is akin to the old design days of lengthy meetings and whiteboard sessions, 
but instead of a team of people, I am using an agent. 

It starts with `STONKS_ACCESS_1.md` which is a conversation with an AI about the idea behind the Stonks project.  I chose to 
use multiple agents in this case, mostly as an experiment to compare the different agents.  The conversation with the agents 
resulted in an aggregated design and feature set.  I repeated this process a couple of times until I felt confident in the 
design and direction of the project.  

I combined these assessments into a single design spec called `POC_1.md`.  The POC was built using claude and this spec file.

Once the technical concepts were proven in the POC, the process repeated for building the MVP.  The MVP files do not 
document the entire process, of iterating over several versions with an agent, but they do demonstrate how the use of spec like files 
produces a clear design.

Overall, this iterative spec driven interactions with an agent has resulted in a decent MVP.
